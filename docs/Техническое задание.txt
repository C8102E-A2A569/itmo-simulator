Тема ВКР
Разработка многопользовательского двумерного симулятора главного корпуса университета ИТМО

Основные вопросы, подлежащие разработке
Разработать многопользовательский двумерный симулятор главного корпуса университета ИТМО — клиент-серверное приложение, объединяющее элементы геймификации, социального взаимодействия и ориентирования по инфраструктуре корпуса. Система должна обеспечивать: регистрацию и аутентификацию пользователей; перемещение игрового персонажа по карте нескольких этажей корпуса с синхронизацией позиций подключённых игроков; отображение плана текущего этажа с указанием положения игрока и ключевых локаций; систему чата между игроками; сохранение и восстановление состояния игрока между сеансами; взаимодействие с объектами окружения.
Клиентская часть должна быть реализована на игровом движке Godot 4 с использованием языка GDScript. Серверная часть должна быть реализована на языке Go. Для поддержки постоянных соединений использовать библиотеку Gorilla WebSocket. Для бинарной сериализации сообщений между клиентом и сервером — Protocol Buffers. Для хранения данных пользователей и состояния игроков использовать PostgreSQL. Развёртывание осуществить посредством Docker. 

Исходные данные
Мобильное приложение itmo.map как источник схем этажей; главный корпус университета ИТМО как объект моделирования; существующие многопользовательские двумерные платформы как объекты сравнительного анализа.


Цель
Расширение набора цифровых инструментов ИТМО путем создания интерактивного многопользовательского симулятора, обеспечивающего игровое дистанционное знакомство с главным корпусом.

Задачи
1.	Провести сравнительный анализ существующих многопользовательских двумерных платформ и обосновать выбор технологического стека.
2.	Разработать программную архитектуру приложения, базы данных и протокол взаимодействия.
3.	Реализовать серверную часть на Go: синхронизацию состояния игроков, обработку взаимодействий с объектами окружения, систему регистрации, аутентификации и сохранения состояния игрока между сеансами.
4.	Реализовать клиентскую часть на игровом движке Godot 4: многоэтажную карту корпуса, механики перемещения персонажа, систему чата, отображение плана этажа и взаимодействие с объектами окружения.
5.	Провести функциональное тестирование и оценить соответствие системы установленным требованиям.

Перечень подлежащих разработке вопросов
1.	Анализ предметной области и обоснование выбора технологического стека.
2.	Проектирование архитектуры системы и протокола клиент-серверного взаимодействия.
3.	Проектирование схемы базы данных.
4.	Разработка серверной части на языке Go с применением библиотеки Gorilla WebSocket.
5.	Разработка клиентской части на игровом движке Godot 4 с использованием языка GDScript.
6.	Функциональное тестирование системы.

Ожидаемый результат
Функционирующее клиент-серверное приложение, обеспечивающее: регистрацию и аутентификацию пользователей; перемещение персонажа по многоэтажной карте корпуса с синхронизацией позиций подключённых игроков; отображение плана текущего этажа; систему чата; сохранение и восстановление состояния игрока между сеансами; взаимодействие с объектами окружения.
Серверная часть реализуется на языке Go с применением библиотеки Gorilla WebSocket, бинарной сериализацией сообщений посредством Protocol Buffers и хранением данных в СУБД PostgreSQL. Клиентская часть реализуется на игровом движке Godot 4 с использованием языка GDScript. Развёртывание системы осуществляется посредством Docker.
